🎨 Visual Editor (Live Preview)

Best for: Content updates where you want to see changes in real-time

✅ Pros
  • Live preview of changes
  • Click-to-edit sections
  • Visual diff before saving
  • Mobile/desktop preview toggle
  • Save multiple changes at once
❌ Cons
  • Requires authentication
  • More complex interface
  • Limited to predefined sections
  • Needs modern browser

📋 How to Use:

1 Access: Go to /admin/visual-editor and login
2 Select Page: Choose page from dropdown (Home, About, Portfolio, Contact)
3 Edit Sections: Click section headers to expand and edit content
4 Preview: See changes in real-time in the right panel
5 Save: Click "Save All" to commit all changes to GitHub
🎨 Open Visual Editor Requires Login Live Preview

⚙️ Custom Editor (Code-Focused)

Best for: Direct HTML editing, YAML data files, and technical modifications

✅ Pros
  • Full control over HTML/CSS
  • Edit YAML data files
  • Create new pages
  • Azure AD authentication
  • GitHub integration
  • Syntax highlighting
❌ Cons
  • Requires HTML knowledge
  • No live preview
  • More technical interface
  • Authentication required

📋 How to Use:

1 Access: Go to /admin/editor and login
2 Choose Task: Edit existing page, edit data files, or create new page
3 Edit Code: Modify HTML, CSS, or YAML content directly
4 Save: Changes are committed directly to GitHub
5 Deploy: GitHub Actions automatically deploy changes
⚙️ Open Custom Editor Technical Requires Login

📝 Prose.io (Simple & Reliable)

Best for: Quick text edits, mobile editing, and when other methods aren't working

✅ Pros
  • Simple, intuitive interface
  • Works on mobile devices
  • 12+ years of reliability
  • GitHub authentication
  • Markdown preview
  • Always available
❌ Cons
  • No live website preview
  • Basic text editor only
  • Limited file management
  • No syntax highlighting for HTML

📋 How to Use:

1 Access: Go to prose.io
2 Login: Authorize with your GitHub account
3 Find Repository: Navigate to krnch/piuwebsite
4 Edit Files: Click any file to edit (HTML, Markdown, YAML)
5 Save: Changes are committed to GitHub automatically
📝 Open Prose.io Mobile-Friendly Quick & Easy

🐙 GitHub Direct (Emergency Backup)

Best for: Emergency fixes, when all other methods fail, or mobile app editing

✅ Pros
  • Always works (99.9% uptime)
  • Full version history
  • Mobile app available
  • File upload capability
  • Branch management
  • No additional authentication
❌ Cons
  • Basic text editor
  • No live preview
  • Technical interface
  • Requires Git knowledge for advanced features

📋 How to Use:

1 Access: Go to github.com/krnch/piuwebsite
2 Navigate: Browse folders to find the file you need
3 Edit: Click the pencil icon (✏️) to edit any file
4 Commit: Add commit message and click "Commit changes"
5 Deploy: Changes deploy automatically via GitHub Actions
🐙 Open GitHub Always Available Mobile App

🎯 When to Use Each Method

📱 Mobile Editing

Best: Prose.io, GitHub app

Why: Touch-friendly interfaces, work offline

🚨 Emergency Fixes

Best: GitHub Direct

Why: Always accessible, no dependencies

📝 Blog Writing

Best: Prose.io or Custom Editor

Why: Simple markdown editing, file management

🎨 Design Changes

Best: Visual Editor, then Custom Editor

Why: Live preview, full HTML control

📊 Data Updates

Best: Custom Editor, then Prose.io

Why: YAML editing, structured data

🔄 Version Control

Best: GitHub Direct

Why: Full git history, branches, pull requests

🚀 Getting Started Recommendations

👤 For Individual Use:

  1. Start with Prose.io - Learn the basics, always reliable
  2. Try Visual Editor - Experience live preview editing
  3. Use GitHub Direct - For emergency situations
  4. Use Custom Editor - When you need full HTML control

👥 For Team Use:

  1. Use GitHub Direct - Version control and collaboration
  2. Train team on Prose.io - Simple, reliable editing
  3. Keep Visual Editor - For technical team members
  4. Maintain GitHub workflow - Pull requests for review

💡 Pro Tips:

  • Bookmark all methods - Different tools for different situations
  • Test changes locally first - Use preview features when available
  • Keep GitHub as backup - It's the most reliable option
  • Use appropriate tool for task - Don't use advanced tools for simple edits
